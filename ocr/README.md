原来的ocr文件夹里太乱了，大量重复的图像操作分散在不同库里，代码结构很乱，需要在这个新文件夹内重构

希望src/main.cpp中尽量简短，结构清晰，主要负责调用ocr_image、ocr_det、ocr_rec库的函数实现识别的流程

# 库的功能设计
+ ocr_image：主要来源于letterbox库，用于处理图像的预处理操作
    + 用于处理所有关于图像的操作
    + 该库使用opencv库来实现图像处理功能
    + BGR格式图像的结构体
        + 图像宽
        + 图像高
        + 图像数据指针
        + 图像数据大小
    + 内部函数：将图像转换为BGR格式，存在上面的结构体中
    + 裁剪+等比缩放图像，输入BGR图像结构体，以及原始图像的x1y1x2y2坐标，以及希望输出的图像尺寸，输出裁剪后的BGR图像结构体
        + 如果x2y2坐标超出图像范围，自动添加黑色像素填充到超出部分
    + 图像输出函数，输入BGR图像结构体，以及输出的图像文件路径，将图像保存到指定路径，保存为png格式
    + 做一个类，类中所有函数都是围绕一个输入的图像进行操作：
        + 初始化的时候输入图像地址，自动转换为BGR格式
        + 输入裁剪后的尺寸和重叠尺寸，自动将BGR格式后的图像重叠裁剪为多个图像块，输出这个图像块的结构体数组，每个图像块要有位置信息，方便后面拼起来
        + 调用ocr_det库里面的函数计算每一个分块文本的位置/热力图信息，然后根据热力图信息和位置信息，将图像块拼起来，形成一个完整图像的热力图，本库里希望留一个函数输入热力图位置信息的结构体数组来拼装完整的热力图
        + 计算热力图的长方形边界，我会输入一些参数，比如热力图的阈值，最小面积，边界扩展系数（纵向扩展比例，横向自动扩展和纵向相同的像素数量），输出这些长方形（x1y1x2y2）的结构体数组
        + 输入上面的那个结构体数组，裁切长方形的图像块，输出这些图像块的结构体数组
        + 图像块的预处理，等比缩放图像块到指定尺寸，如果某些文本图像块过长，重叠裁剪图像块，确保所有文本占满高度，输出这些图像块的结构体数组，注意过长的文本块在裁剪后要做标记，方便后面对识别出来的文字做拼接
        + 析构函数，释放类中所有图像数据指针指向的内存空间
+ ocr_det：主要来源于ocr_det库，用于处理文本检测操作
    + 一个类
        + 初始化的时候输入模型位置，自动加载模型
        + 析构函数，释放模型占用的内存空间
        + 输入BGR图像结构体，输出热力图结构体
+ ocr_rec：主要来源于ocr_rec库，用于处理文本识别操作
    + 一个类
        + 初始化的时候输入模型位置和字典位置，自动加载模型
        + 析构函数，释放模型占用的内存空间
        + 输入BGR图像结构体，输出识别结果结构体
